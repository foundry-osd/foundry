# Cutover Readiness Checklist

- [ ] `Foundry` WinUI app builds in CI for `x64`.
- [ ] `Foundry` WinUI app builds in CI for `ARM64`.
- [ ] `Foundry.Connect` remains WPF and builds successfully.
- [ ] `Foundry.Deploy` remains WPF and builds successfully.
- [ ] All relevant unit tests pass.
- [ ] Velopack package builds.
- [ ] MSI installer builds.
- [ ] Installed app launches.
- [ ] Manual update check works.
- [ ] No-update state works.
- [ ] Logs are created under the expected directory.
- [ ] App settings are created under `C:\ProgramData\Foundry\Settings`.
- [ ] No WinUI `Foundry` runtime data is written under `%LocalAppData%`.
- [ ] Existing expert config can be imported.
- [ ] Deploy config generated by WinUI app is accepted by `Foundry.Deploy`.
- [ ] Generated WinPE media launches `Foundry.Connect`.
- [ ] Generated WinPE media can launch `Foundry.Deploy`.
- [ ] Generated ISO/USB media uses the expected `X:\Foundry`, `Runtime`, `Config`, `Seed`, `Tools`, `OperatingSystem`, and `DriverPack` layout.
- [ ] Generated ISO/USB media stores both runtime agents under `Runtime\<ApplicationName>\<rid>`.
- [ ] WinUI `Foundry` writes new host-side data only to the approved `C:\ProgramData\Foundry` layout.
- [ ] Release workflow publishes all expected artifacts.
- [ ] Main `Foundry` release artifacts are Velopack package/MSI artifacts, not old single-file portable `.exe` assets.
- [ ] `Foundry.Connect` and `Foundry.Deploy` ZIP asset names remain compatible with bootstrap/update logic.
- [ ] Release tags use `vYY.M.D.Build` and Velopack package versions use `YY.M.D-build.Build`.
- [ ] Scheduled release remains disabled until first manual WinUI release succeeds.
